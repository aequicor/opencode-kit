### Свод правил: Как писать навыки, которые работают

| Категория | Правило / Принцип                                                                                                                        | Почему это важно |
| :--- |:-----------------------------------------------------------------------------------------------------------------------------------------| :--- |
| **Триггеры (Triggering)** | **Будьте предельно конкретны.** Описание должно четко указывать сценарии: «Используй этот навык ТОЛЬКО для редактирования файлов .docx». | Агент видит только название и описание при выборе инструмента. Размытые описания (типа «помощь с текстом») ведут к ошибкам выбора. |
| **Архитектура** | **Прогрессивное раскрытие.** Основной файл навыка (`.md`) должен быть кратким (~500 строк), ссылаясь на другие файлы с деталями.         | Большие навыки «забивают» контекстное окно. Компактная структура позволяет модели фокусироваться на текущем шаге. |
| **Экономия токенов** | **Никакой «воды» и очевидности.** Не объясняйте ИИ, что такое Python или SQL. Пишите только то, что специфично для вашей системы.        | Лишний текст конкурирует с полезными данными задачи. Чем меньше токенов тратится на навык, тем качественнее результат. |
| **Примеры (Examples)** | **Включайте исполняемый код.** Описывайте не «на словах», а через конкретные примеры вызовов API или JSON-структур.                      | Наличие примеров кода в навыке повышает вероятность успеха в 3 раза по сравнению с чисто текстовыми инструкциями. |
| **Версионность** | **Точные версии ПО.** Вместо «библиотека React» пишите «React 18.2.0 с Vite и Tailwind».                                                 | ИИ часто галлюцинирует устаревшими методами. Четкая версия направляет его к правильному синтаксису. |
| **Источник знаний** | **Пишите на основе «боли».** Лучший навык — это задокументированный ответ на ошибку, которую агент уже совершил ранее.                   | Навыки, написанные самим ИИ «из головы», часто бесполезны. Навык должен содержать человеческую экспертизу, которой нет у модели. |
| **Обработка ошибок** | **Инструкции по исправлению.** Если скилл возвращает ошибку, она должна содержать совет: «Попробуй формат ГГГГ-ММ-ДД».                   | Это создает цикл обратной связи, позволяя агенту самокорректироваться, а не «галлюцинировать» оправдания. |
