# Migration Guide: 1.2.0 → 1.3.0

## Breaking Changes

### Provider ID format changed
Provider IDs now use underscores instead of hyphens in generated `opencode.json`.
This is auto-handled by the apply script — no manifest changes needed.

### RBAC in bash permissions
`opencode.json.template` now includes additional security constraints for Docker,
Kubernetes, and Terraform commands.

## Upgrade steps

### 1. Run apply with --merge

```bash
python scripts/apply.py \
  --manifest my-project.yaml \
  --target /path/to/project \
  --merge
```

`--merge` overwrites only kit-managed files (agents, skills, templates) while preserving
user-created documents.

### 2. Review the changes

Files changed:
- `opencode.json` — added infrastructure security rules
- `.opencode/_shared.md` — added HITL approval gates
- Agent files — updated frontmatter format

### 3. Update your manifest

Add new optional fields to your manifest:

```yaml
# Multi-provider support (Phase 3)
provider:
  name: routerai
  base_url: https://routerai.ru/api/v1
  api_key_env: ROUTERAI_OPENCODE

  # NEW: Per-agent provider overrides (optional)
  agents:
    CodeWriter:
      provider: openai
      base_url: https://api.openai.com/v1
      api_key_env: OPENAI_API_KEY
```

### 4. Verify

```bash
python -m pytest tests/
```

### 5. New capabilities

1. **HITL gates** — destructive operations require PO approval
2. **Schema validation** — `jsonschema validate my-manifest.yaml manifest.schema.json`
3. **Telemetry** — agent runs are tracked with timing and token counts
4. **New profiles** — `rust-actix`, `go-gin`, `java-spring`, `csharp-aspnet`

# Migration Guide: 1.0.0 → 1.1.0

## Breaking: tools → permission

All agent templates migrated from deprecated `tools` field to `permission`.

### Before (1.0.0)

```yaml
tools:
  read: allow
  edit: allow
```

### After (1.1.0+)

```yaml
permission:
  read: allow
  edit: allow
```

**Action:** Re-run `apply.py --merge` to regenerate agent files.
